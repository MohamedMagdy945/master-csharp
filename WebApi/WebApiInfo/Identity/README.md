# 🚀 ASP.NET Core Identity Web API

A complete **ASP.NET Core Web API** demonstrating a full authentication and authorization system using **ASP.NET Core Identity** and **Entity Framework Core**.

---

## 📌 Overview

This project implements a secure authentication system with:

* Custom `ApplicationUser` with extra profile fields
* Identity and Role management
* JWT Bearer authentication
* Role-based authorization

---

##  Domain & Identity Models

### Base IdentityUser

```csharp
public class IdentityUser<T>
{
    public T Id { get; set; }
    public string Name { get; set; }
    public string Email { get; set; }
    public string PasswordHash { get; set; }
    public bool IsReadOnly { get; set; }
}
```

### Custom IdentityUser

```csharp
public class IdentityUser : IdentityUser<string>
{
    public string ID { get; set; }

    public IdentityUser()
    {
        ID = Guid.NewGuid().ToString();
    }
}
```

### ApplicationUser (Extended User)

```csharp
public class ApplicationUser : IdentityUser
{
    public string Address { get; set; }
    public string Phone { get; set; }
    public string SSN { get; set; }
    public int Grade { get; set; }
}
```

### DbContext — CompanyContext

```csharp
public class CompanyContext : IdentityDbContext<ApplicationUser>
{
    public DbSet<Employee> Employees { get; set; }
    public DbSet<Department> Departments { get; set; }

    public CompanyContext(DbContextOptions<CompanyContext> options)
        : base(options) { }

    protected override void OnModelCreating(ModelBuilder builder)
    {
        base.OnModelCreating(builder);
        // additional configuration
    }
}
```

## Authentication & Authorization Workflow

1. Client sends HTTP request with JWT in `Authorization: Bearer <token>` header.
2. JWT Bearer middleware validates token and extracts claims.
3. Identity system maps claims to `ApplicationUser`.
4. Authorization checks roles and policies.
5. Server responds based on authentication and authorization result.

### Claims Included in JWT

| Claim    | Description            |
| -------- | ---------------------- |
| UserId   | Unique user identifier |
| UserName | Username               |
| Email    | User email             |
| Role     | Role(s) of the user    |
| Grade    | Custom grade claim     |

AspNetUserRoles	Maps users to roles
AspNetUserClaims	Stores claims per user
AspNetRoleClaims	Stores claims per role
AspNetUserLogins	Stores external login info (Google, Facebook)
AspNetUserTokens	Stores refresh tokens or other token-based info
Custom Domain Tables
Table Name	Purpose
Employees	Stores company employees info

## Best Practices & Security

* Use **built-in `PasswordHasher`** for password hashing.
* Require **email confirmation** before granting full access.
* Enforce **strong password policies**.
* Use **HTTPS** for all requests.
* Define **role-based policies** and permissions.
